# 노션 기반 견적서 웹 서비스 MVP PRD

## 핵심 정보

**목적**: 발급자가 노션에 입력한 견적서를 클라이언트가 웹 브라우저에서 바로 확인하고 PDF로 다운로드할 수 있도록 한다.
**사용자**: 견적서를 발행하는 프리랜서/소규모 사업자, 그리고 견적서를 받아 확인하는 클라이언트(고객)

---

## 사용자 여정

```
[발급자 여정]

1. 노션 데이터베이스
   ↓ 견적서 페이지 작성 (품목, 수량, 단가, 고객 정보 등 입력)

2. 노션 페이지 공개 설정 불필요
   ↓ 웹 서비스가 Notion API로 데이터 읽기
   ↓ 견적서 고유 링크 생성됨 (노션 페이지 ID 기반)

3. 링크 전달
   ↓ 발급자가 클라이언트에게 견적서 URL 공유

---

[클라이언트 여정]

1. 견적서 뷰어 페이지 접속 (공유 링크)
   ↓ 노션 API로 견적서 데이터 조회

   [유효한 견적서] → 견적서 뷰어 페이지 → PDF 다운로드 옵션
   [존재하지 않음] → 오류 안내 페이지

2. 견적서 내용 확인
   ↓ 발급자/고객 정보, 품목 목록, 합계 금액 확인

3. PDF 다운로드 버튼 클릭
   ↓ 견적서 PDF 파일 생성

4. 완료 → PDF 저장 또는 인쇄
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 견적서 조회 | Notion API로 특정 견적서 데이터를 읽어 웹에 렌더링 | 서비스의 유일한 핵심 기능 | 견적서 뷰어 페이지 |
| **F002** | 견적서 항목 표시 | 품목명, 수량, 단가, 금액 합계, 세금, 발급일, 유효기간, 발급자/고객 정보를 구조화된 레이아웃으로 표시 | 견적서 내용 전달의 핵심 | 견적서 뷰어 페이지 |
| **F003** | PDF 다운로드 | 현재 견적서 뷰어를 PDF 파일로 변환하여 다운로드 | 클라이언트가 문서를 보관·전달할 수 있는 기본 기능 | 견적서 뷰어 페이지 |
| **F004** | 견적서 목록 조회 | Notion 데이터베이스에서 전체 견적서 목록을 읽어 표시 | 발급자가 발행한 견적서를 관리하는 기본 기능 | 견적서 목록 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | Notion API 연동 설정 | Notion Integration Token과 데이터베이스 ID를 환경 변수로 관리, API 요청/응답 처리 | 모든 핵심 기능의 데이터 소스 | 서버 사이드 (API 레이어) |

## 핵심 구현 참고사항

### Notion API 연동 흐름

```
클라이언트 요청
  → Next.js Route Handler (app/api/invoices/route.ts)
  → @notionhq/client로 Notion API 호출
  → Notion 응답 데이터를 Invoice 도메인 모델로 매핑
  → 클라이언트에 JSON 응답
```

### 품목 데이터 저장 방식

Notion은 관계형 하위 테이블을 지원하지 않으므로, 품목 목록은 **Notion 페이지 본문(Block)**을 활용합니다.

- 견적서 페이지 본문에 **테이블 블록(Table Block)** 형태로 품목 입력
- API에서 `blocks.children.list`로 테이블 블록 읽기
- 각 행을 `InvoiceItem`으로 변환

### PDF 다운로드 구현 방식

- 뷰어 페이지에서 `window.print()` 활용 (CSS `@media print` 스타일 적용) — 가장 간단한 접근
- 또는 `html2canvas` + `jsPDF`로 클라이언트 사이드 PDF 생성
